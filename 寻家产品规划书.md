# 寻家（Find Home）产品规划书

## 一、项目背景

### 1.1 市场痛点

**购房者决策困境**

- 平均看房周期3-6个月，累计看房20-50套，信息过载严重
- 现有工具割裂：在贝壳/链家收藏房源 → 实地看房 → 用Excel/备忘录记录 → 回家手动对比
- **关键信息流失**：实地发现的"主卧西晒"、"楼下广场舞噪音"、"房东急售可砍价"等主观信息，无法沉淀在现有平台

**数据自主权缺失**

- 大平台房源数据不可编辑，用户发现信息错误（面积虚标、楼层错误）无法修正
- 收藏夹只是"浏览历史"，不具备结构化管理和分析能力

### 1.2 用户行为验证

- 36氪报道：购房者"每次看完盘以后，都会列出几个盘的优劣势，然后制作成excel表格上传到云笔记"
- 小红书/知乎大量"看房日记"内容，证明用户有强烈记录和分享需求
- 现有通用工具（Notion/Excel）无法满足房源信息关联、户型图管理、AI分析等垂直场景

## 二、产品定位

| 维度 | 定义 |
| --- | --- |
| **一句话定位** | 寻家，让每一次看房都有迹可循 |
| **产品类型** | 购房决策辅助工具（可编辑的房源对比表格，替代Excel/备忘录） |
| **核心价值** | 结构化房源数据管理 + 可视化对比 + AI智能分析 |
| **目标用户** | 一二线城市25-40岁刚需/改善型购房者 |
| **使用场景** | 看房信息录入、多房源表格对比、看房记录沉淀、辅助决策 |

## 三、核心优势

### 3.1 差异化竞争力

| 优势 | 具体说明 | 竞品对比 |
| --- | --- | --- |
| **表格化管理** | 主界面即大表格，所有房源一览无余，支持排序/筛选/自定义列，房号显示在小区名旁 | Excel无结构化模板，贝壳不可编辑 |
| **数据可编辑** | 编辑模式下表格内直接修改，支持添加自定义字段 | 贝壳/链家仅支持浏览 |
| **结构化备注** | 预设看房维度标签（户型/位置/价格/装修），支持照片+文字 | 备忘录无法关联房源 |
| **可视化对比** | 勾选2-3套一键对比，关键字段对齐，差异高亮 | 大平台仅支持硬性指标对比 |
| **AI分析建议** | 自动生成优缺点分析、适合人群、议价建议 | 大平台无个性化分析 |
| **截图识别录入** | 上传贝壳/链家截图，AI自动提取房源数据一键录入，截图自动设为封面 | 手动录入费时费力 |
| **隐私优先** | 个人数据不与平台共享 | 大平台收藏行为用于算法推荐 |

### 3.2 产品形态

核心形态为**可编辑的专业表格**，而非信息流或卡片列表：

- **列表模式**：全屏大表格展示所有房源，支持排序、搜索、标签筛选
- **编辑模式**：表格内直接编辑数据，支持添加/删除自定义列
- **详情面板**：点击某行展开右侧详情（基础信息 + 看房时间轴 + AI分析）
- **对比模式**：勾选多套房源，右侧展示对比表格，差异高亮（优/劣箭头标识）

### 3.3 商业模式优势

- **避开红海**：不做房源平台，而是替代Excel做购房决策工具
- **数据合规**：用户手动录入数据，无爬虫风险
- **高粘性场景**：购房决策周期长（3-6个月），用户留存天然高于普通工具类App

## 四、产品功能架构

```
寻家 Find Home
├── 房源管理层（核心 - 大表格）
│   ├── 表格列表（排序/筛选/搜索/自定义列/列显隐配置）
│   ├── 表格内编辑（编辑模式直接修改数据）
│   ├── 添加/删除房源
│   ├── 截图识别录入（上传截图 → AI提取 → 直接入库无闪烁，含房号和自定义列）
│   └── 收藏标记 + 状态管理（已看/待看/已售）
├── 看房记录层
│   ├── 结构化备注（分类标签：户型/位置/价格/装修）
│   ├── 照片记录（看房现场拍照）
│   └── 时间轴回顾（历次看房记录）
├── 对比分析层
│   ├── 多房源对比表格（2-3套并排，差异高亮）
│   ├── AI分析报告（优缺点/适合人群/议价建议）
│   └── 决策建议
└── 后续扩展
    ├── 数据持久化（Supabase 云端存储）
    ├── 用户认证（多设备同步）
    ├── 导出Excel/PDF
    └── 看房路线规划
```

## 五、目标与里程碑

| 阶段 | 时间 | 目标 |
| --- | --- | --- |
| **MVP** | 1-2个月 | 前端表格+编辑+对比+详情，本地Mock数据验证核心交互 |
| **V1.0** | 3-4个月 | 接入Supabase数据库+用户认证，数据持久化，图片上传，上线部署 |
| **V1.5** | 4-6个月 | 截图识别自动录入、AI对比分析、导出功能、移动端适配 |
| **增长期** | 6-12个月 | 通过小红书/知乎内容营销获取种子用户，验证付费意愿 |

## 六、风险评估

| 风险 | 应对策略 |
| --- | --- |
| 大平台抄袭功能 | 专注个人数据管理+可编辑表格，与平台商业模式错位 |
| 用户录入门槛 | 提供截图识别自动录入，大幅降低录入成本；同时保留手动添加入口 |
| 数据准确性争议 | 所有数据为用户自主录入和编辑，明确标注数据来源 |
